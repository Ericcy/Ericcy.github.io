分支

多任务并行开发的时候，分支可以非常方便的帮助我们管理代码！

每进行一个新的任务，就开辟一个分支。

在分支下进行的提交操作是将版本生成到当前分支！！

在切换分支之前，一定要将所有的修改内容做一次提交！

远程仓库

远程仓库主要用来做版本库共享！！ 多人协作开发，使用远程仓库

远程仓库其实就是一个裸仓库

裸仓库： 将所有文件都裸露在外面，并没有.git文件夹

一般在push之前都会先pull一下！！

一般用作服务的远程管理！！

ssh登录服务器

秘钥的生成

服务器的免密登录

  1. 生成公钥以及私钥

  2. 复制公钥中所有的内容

  3. 登录服务器（要使用密码登录）

  4. 打开服务器上的根目录下面的.ssh文件夹 编辑authorized_keys文件

  5. 将复制的公钥 全部粘贴到这个文件下面即可

  6. 重新登录即已实现免密！

github免密登录

  1. 生成公钥以及私钥

  2. 复制公钥中所有的内容

  3. 打开github个人设置页面

  4. 找到ssh and gpg keys

  5. 点击 New SSH Key按钮

  6. 将复制的公钥内容粘贴到 key 文本框中

  7. 点击Add SSH Key 按钮

  8. 实现了github免密功能了

  9. 只有使用ssh连接才能够免密！！！！！
